Le référencement

Les moteurs de recherche fonctionnent grâce à trois fonctions principales :

  1. Crawling: Ils parcourent l'internet à la recherche de contenu, en examinant le code/contenu de chaque URL qu'ils trouvent.
  2. Indexing: Stocker et organiser le contenu trouvé au cours du processus de crawling. Une fois qu'une page est dans l'index, elle est dans la course pour être affichée comme résultat aux requêtes pertinentes.
  3. Ranking: Fournir les éléments de contenu qui répondront le mieux à la requête d'un chercheur, ce qui signifie que les résultats sont classés du plus pertinent au moins pertinent.
Le référencement - Crawling

Le crawling est le processus de découverte au cours duquel les moteurs de recherche envoient une équipe de robots (appelés crawlers ou spiders) pour trouver du contenu nouveau et mis à jour.

Le référencement - Indexing

Les moteurs de recherche traitent et stockent les informations qu'ils trouvent dans un index, une énorme base de données de tout le contenu qu'ils ont découvert et qu'ils jugent suffisamment bon pour être proposé aux chercheurs.

Le référencement - Ranking

Lorsqu'une personne effectue une recherche, les moteurs de recherche parcourent leur index à la recherche de contenu hautement pertinent et ordonnent ensuite ce contenu dans l'espoir de répondre à la requête de l'internaute.

Le référencement - Google Bot

Les erreurs de navigation courantes qui peuvent empêcher les robots d'indexation de voir l'intégralité de votre site :

  1. La navigation mobile affiche des résultats différents de ceux de la navigation de bureau.
  2. Tout type de navigation où les éléments du menu ne se trouvent pas dans le code HTML, comme les navigations en JavaScript.
  3. La personnalisation, ou l'affichage d'une navigation unique pour un type spécifique de visiteur par rapport aux autres.
  4. Oublier de créer un lien vers une page principale de votre site Web par le biais de votre système de navigation - n'oubliez pas que les liens sont les chemins que suivent les robots d'exploration pour accéder aux nouvelles pages !
Le référencement - Robots.txt

Les fichiers Robots.txt sont situés dans le répertoire racine des sites web et suggèrent les parties de votre site que les moteurs de recherche doivent ou ne doivent pas explorer, ainsi que la vitesse à laquelle ils explorent votre site.


Comment Googlebot traite-t-il les fichiers robots.txt ?

  1. Si Googlebot ne trouve pas de fichier robots.txt pour un site, il procède à l'exploration du site.
  2. Si Googlebot trouve un fichier robots.txt pour un site, il se conformera généralement aux suggestions et procédera à l'exploration du site.
  3. Si Googlebot rencontre une erreur en essayant d'accéder au fichier robots.txt d'un site et ne peut pas déterminer s'il existe ou non, il n'explorera pas le site.

Asynchronous JavaScript - AJAX

AJAX signifie Asynchronous JavaScript and XML. L'AJAX n'est pas un langage de programmation mais correspond plutôt à un ensemble de techniques utilisant des technologies diverses pour envoyer et récupérer des données vers et depuis un serveur de façon asynchrone, c'est-à-dire sans avoir à recharger la page

“L'AJAX” ou plutôt “l'Ajax” est aujourd'hui un terme générique utilisé pour désigner toute technique côté client (côté navigateur) permettant d'envoyer et de récupérer des données depuis un serveur et de mettre à jour dynamiquement le DOM sans nécessiter l'actualisation complète de la page.

HTTP Protocol detour

Les serveurs Web parlent le protocole HTTP, c'est pourquoi ils sont souvent appelés serveurs HTTP. Ces serveurs HTTP stockent les données de l'Internet et fournissent les données lorsqu'elles sont demandées par les clients HTTP.

HTTP Protocol detour - Media Types

Comme Internet héberge plusieurs milliers de types de données différents, HTTP marque soigneusement chaque objet transporté sur le Web avec une étiquette de format de données appelée type MIME.

• Un document texte au format HTML serait étiqueté avec le type text/html

• Un document de texte ASCII simple serait étiqueté avec le type text/plain

• Une version JPEG d'une image serait image/jpeg

• Une image au format GIF serait image/gif

• Un film Apple QuickTime serait video/quicktime.

• Une présentation Microsoft PowerPoint serait application/vnd.mspowerpoint

HTTP Protocol detour - URLs

Les URL décrivent l'emplacement spécifique d'une ressource sur un serveur particulier. Elles vous indiquent exactement comment aller chercher une ressource à partir d'un emplacement précis et fixe.


HTTP Protocol detour - Methods

Le protocole HTTP prend en charge plusieurs commandes de requête différentes, appelées méthodes HTTP. Chaque requête HTTP possède une méthode. La méthode indique au serveur l'action à effectuer


HTTP methodDescription
GETEnvoyer une ressource nommée du serveur au client
POSTEnvoyer les données du client dans une application passerelle serveur
PUTStocker les données du client dans une ressource serveur nommée
DELETESupprime la ressource nommée d'un serveur.
HEADEnvoyer uniquement les en-têtes HTTP de la réponse pour la ressource nommée.
HTTP Protocol detour - Status Codes

Chaque message de réponse HTTP est accompagné d'un code d'état. Ce code d'état est un code numérique à trois chiffres qui indique au client si la demande a abouti ou si d'autres actions sont nécessaires.

Asynchronous JavaScript - XMLHttpRequest

Les objets XMLHttpRequest (XHR) permettent d'interagir avec des serveurs. On peut récupérer des données à partir d'une URL sans avoir à rafraîchir complètement la page. Cela permet à une page web d'être mise à jour sans perturber les actions de l'utilisateur.

Initialement, XMLHttpRequest était utilisé pour la boîte de réception de Gmail. Le fonctionnement était le suivant : toutes les 20 à 30 secondes, une nouvelle requête était envoyée pour savoir si vous aviez ou non du nouveau courrier.

AJAX - XMLHttpRequest

L'objet XMLHttpRequest permet à JavaScript côté client d'émettre des requêtes HTTP et de recevoir des réponses (qui ne doivent pas nécessairement être XML) des serveurs Web.

  1. Créez un objet XMLHttpRequest avec le constructeur XMLHttpRequest().
  2. Appelez open() pour spécifier l'URL et la méthode (généralement "GET" ou "POST") de la requête.
  3. Définissez la propriété onreadystatechange à la fonction qui sera notifiée de la progression de la demande.
  4. Définissez la propriété onreadystatechange à la fonction qui sera notifiée de la progression de la demande.
  5. Appelez setRequestHeader(), si nécessaire, pour spécifier des paramètres de requête supplémentaires.

JSON.stringify()

JSON est utilisé pour la transmission et le stockage de données structurées. Il sert de format de données lors de la transmission de données

Valeur nulle: est représenté par le mot-clé null.

Valeur booléenne: est représentée par les mots-clés true et false.

Nombre: est une suite de chiffres de 0 à 9. Cette suite peut être introduite par un signe négatif - et interrompue par un point décimal .

Chaîne de caractères: commence et se termine par des guillemets droits doubles (")

Array: commence par [ et se termine par ]. Il contient une liste indexée d'éléments de même type ou de types différents, séparés par des virgules. Les tableaux vides sont autorisés.

Object: commence par { et se termine par }. Il contient une liste de propriétés non ordonnée, divisée par des virgules. Les objets sans propriétés ("objets vides") sont autorisés.

JSON.stringify()

Tout objet JavaScript peut être stringifié (converti en chaîne de caractères) à l'aide de la fonction JavaScript JSON.stringify().
